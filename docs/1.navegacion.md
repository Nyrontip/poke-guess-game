# 🧭 Navegación — Poke Guess Game

## Flujo principal

El juego es una SPA (Single Page Application) construida con Solid.js.
No usa router externo — la navegación se maneja con un estado `route` y componentes `Show` en `App.jsx`.

```
Home
  │
  ├── [Select Player] ──────────────┐
  │                                 │
  └── [New Player] → Register ──────┤
                                    ▼
                                ┌─────────┐
                                │  GAME   │
                                │ Q 1..10 │
                                └────┬────┘
                                     │
                          ┌──────────┴──────────┐
                          │                     │
                    [Finish 10]            [Quit]
                          │                     │
                          ▼                     ▼
                     ┌─────────┐           ┌────────┐
                     │ RESULTS │           │  HOME  │
                     │ Score   │           └────────┘
                     │ Board   │
                     └────┬────┘
                          │
                    ┌─────┴─────┐
                    │           │
                 [Retry]    [Home]
                    │           │
                    ▼           ▼
                ┌────────┐ ┌────────┐
                │  GAME  │ │  HOME  │
                └────────┘ └────────┘
```

## Pantallas

### Home (`/` — ruta `'home'`)
- Muestra lista de jugadores existentes
- Botón "New Player" para registrar nuevo
- Botón 📊 por jugador para ver historial
- **Sub-flujo History**: cambia `mode()` a `'history'` y muestra scores del jugador con filtros Week / Month / All Time. Botón "Back" vuelve a la lista.

### Game (`/game` — ruta `'game'`)
- 10 preguntas secuenciales, cada una con timer de 12 segundos
- 6 opciones por pregunta
- Al responder o agotar el tiempo, avanza automáticamente tras 800ms
- Botón "Quit" vuelve a Home sin guardar

### Results (`/results` — ruta `'results'`)
- Muestra resumen: `correctos/10` y tiempo total
- Botón "Save Score" → persiste y refresca el leaderboard
- Tabla Top 5 scores
- Botones "Retry" (vuelve a Game) y "Home" (vuelve a Home)

## Estado compartido

El estado `session` en `App.jsxx` contiene:

```ts
{
  player: { id, name, avatar },  // quién juega
  game: { correct, totalTimeMs }  // resultado de la partida
}
```

`session` se pasa como prop a cada página. No hay estado global ni Context API — fluye por props, que para 3 pantallas alcanza.

## Reglas de navegación

- No se puede ir a Results sin haber pasado por Game
- Si el jugador no elige un player ni registra uno, no puede jugar
- El botón "Quit" en Game descarta la partida en curso
- Retry desde Results inicia una nueva partida (mismo jugador)
- Home desde Results vuelve a la selección de jugador

## Diagrama de dependencias entre componentes

```
App.jsx
 ├── Home.jsx     ← solo recibe `onStart(player)`
 ├── Game.jsx     ← recibe `session`, `onFinish(result)`, `onQuit()`
 └── Results.jsx  ← recibe `session`, `onRetry()`, `onHome()`
```
